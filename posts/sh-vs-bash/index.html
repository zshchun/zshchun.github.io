/bin/sh와 /bin/bash의 차이

2022/08/11 · 346 words · Seunghwan Chun

bash shell 대신 /bin/sh를 사용해야 하는 경우가 있다.

sh와 bash의 문법 차이를 살펴보자.

1. [[ ]] 의 유무#

[[ ]] 는 sh POSIX 표준이 아니다.

조건 표현식 [ “$a” = “$b” ] 와 같이 단일 대괄호 [ ] 를 사용하는 편이 좋다.

2. == 와 =#

프로그래밍 언어에서 사용하는 ==는 sh에서 사용할 수 없다. 호환성을 위해 =를 사용하는 편이 좋다.

-eq -gt -le 와 같은 숫자 비교 식은 사용할 수 있다.

3. source의 유무#

sh에는 source가 없다. 대신 . 만을 사용한다.

4. (( )) 계산식#

$(( )) 계산식은 일부 동작하고 (( )) 단일로는 동작하지 않는다.

물론 for ((i=0;i<3;i++)) 와 같은 반복문도 동작하지 않는다.

5. ${ } 변수#

중괄호를 이용한 ${a} 와 ${a%pattern}, ${a#pattern} 와 같은 파라미터 확장은 동작한다.

6. 추가식#

a+=STRING 과 같은 변수 추가는 불가능하다.

a=${a}STRING 과 같이 추가해야 한다.

7. 배열#

a=(a b c) 와 같은 배열은 동작하지 않는다. set을 이용해 비슷하게 사용할 수는 있다.
